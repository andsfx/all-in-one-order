## Test Scenario 1: Amount Mismatch (Fraud Attempt)

### Setup:
- Order ID: test-order-123
- Order Total: 50000 (Rp 50,000)
- Webhook Amount: 1000 (Rp 1,000)
- Difference: 49000 (exceeds 100 tolerance)

### Expected Behavior:
1. Webhook receives PAYMENT_SETTLED event
2. Signature verification passes
3. Order fetched from database
4. Amount validation fails (49000 > 100)
5. Fraud attempt logged to audit_logs:
   - action: 'FRAUD_ATTEMPT'
   - old_value: '50000'
   - new_value: '1000'
   - metadata includes difference, transaction_id
6. Order status remains 'pending_payment'
7. Webhook returns 200 with error JSON:
   {
     "error": "Payment amount mismatch",
     "expected": 50000,
     "received": 1000
   }

### Console Logs:
```
FRAUD ATTEMPT DETECTED: Amount mismatch {
  order_id: 'test-order-123',
  transaction_id: 'TXN-123',
  expected_amount: 50000,
  received_amount: 1000,
  difference: 49000
}
```

### Database State After:
- orders.status = 'pending_payment' (unchanged)
- orders.paid_at = null (unchanged)
- audit_logs: New row with action='FRAUD_ATTEMPT'
